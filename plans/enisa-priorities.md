## ПРИОРИТЕТЫ (привязаны к ENISA критериям)

### 🔴 P0 — БЕЗ ЭТОГО ENISA НЕ ПРОЙДЁТ

#### P0.1: AI-детекция колёс (Computer Vision)
**Зачем для ENISA**: Это core innovation. "Proprietary AI pipeline для автоматического определения и замены колёс"
**Что делать**:
- Интегрировать SAM (Segment Anything) через fal.ai или Replicate
- Пользователь загружает фото → AI находит колёса → показывает маркеры
- Автоматическое размещение выбранного диска в правильном масштабе и позиции
**Результат**: Пользователь загружает фото → одним кликом видит свою машину с новыми дисками

#### P0.2: Backend API
**Зачем для ENISA**: "Один HTML файл" = не стартап. Нужна профессиональная архитектура.
**Что делать**:
- Python FastAPI: /api/wheels, /api/tryon/detect, /api/tryon/apply, /api/orders
- Swagger/OpenAPI документация (автоматическая из FastAPI)
- Docker-ready
**Результат**: Задокументированный API = доказательство технологической зрелости

#### P0.3: Деплой на live URL
**Зачем для ENISA**: Нужен работающий продукт, который можно показать evaluador
**Что делать**:
- Frontend → Vercel (forgedwheels.eu)
- Backend → Railway/Render
- AI service → тот же backend или serverless
**Результат**: Живой URL в бизнес-плане ENISA

#### P0.4: English UI
**Зачем для ENISA**: Рынок EU/USA/China → сайт ДОЛЖЕН быть на английском
**Что делать**: Перевести весь UI на английский (основной), добавить ES/DE
